# Security Policy

## About This Repository

This repository contains PowerShell scripts and code samples published as part of the [systunation.com](https://systunation.com) blog. The scripts are for demonstration and reference purposes and intended to help IT administrators manage devices and identities using Microsoft Intune, Entra ID, and related Microsoft 365 services.

## Supported Versions

This repository does not follow a versioned release cycle. Scripts are updated as blog posts are revised. Always use the latest version of a script from the `main` branch.

## Reporting a Vulnerability

If you discover a security vulnerability in any script or code in this repository — such as credential exposure, privilege escalation risk, injection flaws, or unsafe API usage — please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, use one of the following:

- **GitHub Private Vulnerability Reporting** *(preferred)*: Use the [Report a vulnerability](../../security/advisories/new) button on the Security tab of this repository. This keeps the report confidential until it is resolved.
- **Blog contact**: Reach out via [systunation.com](https://systunation.com).

## What to Include in Your Report

To help investigate and resolve the issue quickly, please include:

- A description of the vulnerability and its potential impact
- The affected script(s) or file(s) with path(s)
- Steps to reproduce or a proof-of-concept (where applicable)
- Any suggested remediation if you have one

## Scope

This policy covers all scripts and code files in this repository. It does not cover third-party tools, Microsoft services, or external dependencies that the scripts may interact with.

## Disclosure Policy

Once a reported vulnerability is confirmed and a fix is applied, a summary may be published in the relevant blog post or as a GitHub Security Advisory, crediting the reporter if they wish.

Thank you for helping keep this community resource safe.
